# データ保持

データ保持は、投稿、編集内容、関係、履歴などを、どの期間・条件で保存し続けるかを定める設計である。

## 分離して考える対象

利用者に見える演出上の喪失と、実際の保存データは分けて扱う必要がある。ゲーム上のラン終了で表示経路や一時権限を失っても、共同知識や編集履歴まで削除するとは限らない。

保存対象ごとに、永続保存、一定期間後の削除、利用者による削除、運営上の保全を区別する。何を残すかだけでなく、誰が閲覧できるかも別に設計する。

## Wikiでの扱い

SampleWikiでは、Markdownを正本とし、JSON索引は再生成可能な派生データとして扱う。このように正本と派生物を分けると、索引を作り直しても内容を失わずに済む。
