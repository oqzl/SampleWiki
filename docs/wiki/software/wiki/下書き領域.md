# 下書き領域

下書き領域は、未完成の内容を公開ページから分離して保存し、確認や共同編集を経て公開へ移すための領域である。

## 目的

未整理のメモ、出典確認前の記述、構成途中のページを直ちに正本へ混ぜずに保持できる。これにより、探索速度を落とさず、公開品質も維持しやすくなる。

ただし、下書きが長期間放置されると重複ページや古い情報が増える。作成者、作成日、公開条件、破棄条件を記録し、定期的に整理する必要がある。

## Wikiでの扱い

[[出典確認]]や[[編集履歴]]と組み合わせると、確認前の記述と公開済み内容を区別できる。[[共同イベント]]では、参加者ごとの作業場所として使い、完成後に既存ページへ統合する運用が考えられる。
