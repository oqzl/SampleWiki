# 編集履歴

編集履歴は、ページへ加えられた変更を時系列で記録し、以前の状態との差分や変更者を確認できる仕組みである。

Wikiでは本文だけでなく、ページ名、配置、リンク先の変更も知識構造へ影響する。そのため、単なるバックアップではなく、変更理由を追跡するための記録として扱う。

## 主な用途

- 変更前後の差分を確認する
- 誤編集や破損から復元する
- ページ統合や移動の経緯を調べる
- 同じ内容の重複追加を防ぐ

## 設計上の注意

自動生成ファイルの更新と本文の編集を区別すると、履歴を読みやすくなる。大きな統合や削除では、何を残し、何を吸収したかが分かるコミットメッセージを付ける必要がある。
