# バッティング

バッティングは、複数のプレイヤーが同じ選択をすると、その選択が無効になる、価値が下がる、または競合処理が発生するメカニクスである。

## 基本構造

各プレイヤーが同時に行動や対象を選び、公開後に重複を判定する。誰とも重ならなかった選択だけが成功する方式が代表的である。

## 面白さの核

強い選択肢ほど他のプレイヤーも選びやすいため、単純に価値の高い対象を選ぶだけでは成功しない。相手の好み、直前の行動、得点状況を読んで、あえて二番手や意外な選択肢を選ぶ判断が生まれる。

[[ドラフト]]と組み合わせる場合、候補の獲得競争を同時選択で解決したり、複数人が狙った候補を場へ戻したりする設計が可能である。

## 設計上の注意

失敗が連続すると進行が停滞しやすいため、重複時にも小さな補償を与える、次回の優先権を与える、候補数を人数より多くするなどの調整が使われる。
