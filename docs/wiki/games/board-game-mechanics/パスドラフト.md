# パスドラフト

パスドラフトは、各プレイヤーが手札や候補の束から一つを選び、残りを隣のプレイヤーへ渡すことを繰り返すドラフト方式である。

選択したものだけでなく、渡したものと後で戻ってくる可能性のあるものまで判断材料になるため、単純な取り合いよりも継続的な情報読みが生まれる。

## 基本構造

1. 各プレイヤーへ同数の候補を配る
2. その中から一つを選ぶ
3. 残りを左または右隣へ渡す
4. 候補がなくなるまで繰り返す

受け渡し方向をラウンドごとに反転すると、特定の隣人だけから候補を受け取り続ける偏りを抑えられる。この設計は[[ドラフト方向]]として独立して検討できる。

## 面白さの核

パスドラフトでは、現在の選択と将来の選択がつながっている。

強い候補を取る、自分の戦略に合う候補を取る、他者へ渡したくない候補を取る、後で戻ると予測した候補を見送る、といった複数の判断が同時に発生する。

候補の流れを記憶する行為は[[ドラフト記憶]]、隣のプレイヤーの選択傾向を読む行為は[[隣接情報]]として整理できる。

## 設計上の注意

候補数が多すぎると初手の比較負荷が高まり、少なすぎると選択が自動化しやすい。また、人数が増えるほど一度手放した候補が戻るまでの時間が長くなり、記憶よりも不確実性が強くなる。

全員が同時に選ぶ方式では待ち時間を抑えられるが、公開情報が少ないと他者の意図を読みづらい。選択後に一部を公開する[[公開ドラフト情報]]を組み合わせると、情報戦を強められる。
