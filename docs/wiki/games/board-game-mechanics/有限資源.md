# 有限資源

有限資源は、ゲーム中に得られる総量や使用回数が制限され、使う時点と配分が重要になる資源である。

## 基本構造

資源が有限であると、プレイヤーは現在の利益と将来の余地を比較する必要がある。残量がそのまま選択肢の広さになるため、消費は単なる支払いではなく進行管理になる。

[[資源消費]]と組み合わせる場合、回復手段の有無、補充時期、他の資源への変換率によって緊張感が変わる。全員で共有する有限資源では、協力と取り合いが同時に発生する。

## 設計上の注意

- 初期量から最終盤までの消費曲線を想定する
- 枯渇後にも最低限の行動手段を残す
- 残量を公開するか隠すかを決める
- 節約だけが常に正解にならない利益を用意する

有限性が強すぎると序盤の失敗が最後まで固定される。回復、代替手段、役割分担によって挽回可能性を調整する必要がある。
