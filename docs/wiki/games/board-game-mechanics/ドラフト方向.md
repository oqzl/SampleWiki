# ドラフト方向

ドラフト方向は、パスドラフトで候補を左回りまたは右回りのどちらへ渡すかを定める設計要素である。

単なる進行方向に見えるが、誰から候補を受け取り、誰へ候補を渡すかを固定するため、プレイヤー間の情報関係や妨害可能性に影響する。

## 基本パターン

- 全ラウンドを同じ方向で進める
- ラウンドごとに左右を反転する
- 特定条件で方向を変更する
- プレイヤーが方向を選択する

同じ方向を維持すると、隣人との関係を継続的に読める。一方、ラウンドごとに反転すると、異なる隣人から候補を受け取るため、局所的な偏りを抑えやすい。

## 情報構造

方向が固定されている場合、上流側のプレイヤーが何を集めているかを推測しやすくなる。この関係は[[ドラフト上流]]と[[ドラフト下流]]として分けて考えられる。

また、下流側へ有力候補を渡さない判断は[[隣接ブロッキング]]につながる。方向が途中で変わる場合は、誰が次の上流になるかを見越す[[方向転換タイミング]]が新たな判断材料になる。

## 設計上の注意

方向変更が頻繁すぎると候補の流れを追いにくくなり、ドラフト記憶の意味が薄れる。反対に固定し続けると、席順による有利不利や、特定の隣人との相性がゲーム全体を支配しやすい。

方向は独立したルールではなく、公開情報の量、候補数、人数、ラウンド数と組み合わせて調整する必要がある。
