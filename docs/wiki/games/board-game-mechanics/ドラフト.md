# ドラフト

ドラフトは、複数の候補から一つを選び、残りを他のプレイヤーへ回す、または共通の場へ残すことで、選択に競争と情報読みを持たせるメカニクスである。

単に強いカードを取るだけではなく、自分に必要なもの、他のプレイヤーが必要としそうなもの、後で戻ってきそうなものを同時に考える点に特徴がある。

## 基本構造

典型的なドラフトでは、各プレイヤーが候補の束から一枚を選び、残りを隣へ渡す。この形式は[[パスドラフト]]として整理できる。

共通の場に公開された候補から順番に選ぶ方式は[[オープンドラフト]]として整理できる。候補が公開されるため、他のプレイヤーの方針を読みやすい一方、手番順の価値が大きくなる。

## 選択の意味

ドラフトの選択には、少なくとも次の三つの意味が重なる。

- 自分の得点や能力を伸ばす
- 他のプレイヤーの狙いを推測する
- 後の選択肢を調整する

他のプレイヤーへ渡したくない候補を先に取る判断は[[カットドラフト]]として整理できる。自分の成長との釣り合いが重要になる。

## 設計上の注意

ドラフトでは、選択肢が多すぎると比較負荷が高まり、少なすぎると判断が自動化しやすい。候補数、公開情報、受け渡す方向、同時選択か順番選択かによって感触が変わる。

また、最初に選べるプレイヤーが恒常的に有利にならないよう、[[ドラフト順序]]の変更、逆順選択、補償資源などを組み合わせる方法がある。
