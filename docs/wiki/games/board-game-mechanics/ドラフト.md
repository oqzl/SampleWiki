# ドラフト

ドラフトは、複数の候補から一つを選び、残りを他のプレイヤーへ渡す、または共通の場へ残すことで、選択に競争と情報読みを持たせるメカニクスである。

単に強い候補を取るだけではなく、自分に必要なもの、他のプレイヤーが必要としそうなもの、後で戻ってきそうなものを同時に考える点に特徴がある。

## 基本形式

### パスドラフト

各プレイヤーへ同数の候補を配り、その中から一つを選んで残りを隣のプレイヤーへ渡す。候補がなくなるまで選択と受け渡しを繰り返す。

選択したものだけでなく、渡したものや後で戻ってくる可能性のあるものも判断材料になるため、現在の選択と将来の選択が連続する。

### オープンドラフト

共通の場に公開された候補から順番に選ぶ方式である。他のプレイヤーの方針を読みやすい一方、手番順の価値が大きくなりやすい。

### カットドラフト

自分の成長よりも、他のプレイヤーへ渡したくない候補を先に取る判断を指す。妨害による利益と、自分向けの候補を見送る損失の釣り合いが重要になる。

## 選択の意味

ドラフトの選択には、主に次の意味が重なる。

- 自分の得点や能力を伸ばす
- 他のプレイヤーの狙いを推測する
- 相手に有力候補を渡さない
- 後の選択肢や候補の流れを調整する

パスドラフトでは、候補の流れを記憶し、隣のプレイヤーの選択傾向を読むことで、どの種類が再び流れてくるか、どの方針で競合しているかを推測できる。ただし偶然の候補配分を意図的な信号と誤認しないよう、複数巡の傾向や公開された獲得物を組み合わせて判断する必要がある。

## 受け渡し方向と隣接関係

パスドラフトでは、候補を左回りまたは右回りのどちらへ渡すかによって、誰から受け取り、誰へ渡すかが決まる。

主な設計パターンには次がある。

- 全ラウンドを同じ方向で進める
- ラウンドごとに左右を反転する
- 特定条件で方向を変更する
- プレイヤーが方向を選択する

方向を固定すると、候補を渡してくる上流側と、候補を渡す下流側との関係を継続的に読みやすい。上流側が特定の種類を集めている場合、その種類は自分まで流れてきにくくなるため、競合の少ない方針へ切り替える判断が生まれる。

一方、方向をラウンドごとに反転すると異なる隣人から候補を受け取るため、局所的な偏りや席順による有利不利を抑えやすい。反転が頻繁すぎると候補の流れを追いにくくなるため、人数、候補数、ラウンド数、公開情報の量と合わせて調整する。

## 設計上の注意

候補が多すぎると比較負荷が高まり、少なすぎると判断が自動化しやすい。人数が増えるほど、一度手放した候補が戻るまでの時間が長くなり、記憶よりも不確実性が強くなる。

全員が同時に選ぶ方式は待ち時間を抑えられるが、公開情報が少ないと他者の意図を読みづらい。獲得物の一部を公開する、選択結果を段階的に公開するなどの方法で情報戦を調整できる。

順番に選ぶ方式では、最初に選べるプレイヤーが恒常的に有利にならないよう、選択順の変更、逆順選択、補償資源などを組み合わせる方法がある。
