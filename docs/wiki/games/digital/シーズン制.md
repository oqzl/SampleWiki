# シーズン制

シーズン制は、一定期間を一つの区切りとして、進行状況、順位、報酬、利用可能な要素などを更新または再編成する方式である。

## 基本構造

シーズンの開始時に共通の初期条件を設け、期間中の活動を蓄積し、終了時に結果を確定する。次のシーズンでは一部の状態を初期化しつつ、実績や恒久的な要素だけを残す構成が多い。

期間による区切りは、新規参加者が途中から入りやすくなる一方、短期間での参加圧力や取り逃し感を生みやすい。更新頻度、持ち越す情報、失われる資源を明確にする必要がある。

## 探索型サービスでの利用

WikiやSNSに導入する場合、表示経路、共同イベント、探索状態などをシーズンごとに変化させられる。本文や共同知識は保持し、探索上の状態だけを再編成すると、蓄積性を損なわずに周回性を作れる。
