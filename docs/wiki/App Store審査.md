# App Store審査

AppleがApp Store Connectへ提出されたアプリ、アップデート、アプリ内購入、アプリ内イベントなどを確認するプロセス。

## 雑学

Appleの公式説明では、すべての提出物を審査し、ユーザーに安全で信頼できる体験を提供することを目的としている。

審査は単なる動作確認ではなく、「安全性」「パフォーマンス」「ビジネス」「デザイン」「法的事項」という複数の観点で行われる。アプリ本体だけでなく、説明文、スクリーンショット、プライバシー情報などのメタデータも、実際の機能を正確に表す必要がある。

Appleは、審査でよく起きる問題としてクラッシュ、壊れたリンク、仮置きコンテンツ、審査に必要な情報の不足などを挙げている。アカウントが必要なアプリでは、デモアカウントや全機能を確認できるデモモードが求められる場合がある。

つまりApp Store審査は、完成したバイナリを最後に検査するだけの工程ではなく、企画段階から考慮する設計条件でもある。
